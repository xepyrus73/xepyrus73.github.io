반응표면 분석의 Response Optimizer



분석을 통하여 얻어진 결과를 토대로 최적의 값을 구하는 최적수준을 구하는 것이 최종의 목적이다. 어떤 수준의 조합인 경우에 최적 값을 갖는지에 대하여 MINITAB에서는 Contour/Surface Plot, Overlaid Contour Plot, Response Optimizer 등의 방법을 제공한다. 이 중에서 Response Optimizer는 반응변수가 하나 이상일 때 반응변수에서 목표하는 값에 만족하는 최적의 인자의 수준조합을 찾아내려고 할 때 사용한다.


1. Response Optimizer의 실행 (데이터 : Central02.mtw)

어떤 화학공장에서 생산되고 있는 제품의 마지막 생산과정에서 제품의 반응량에 영향을 주는 3가지 인자 (교반속도, 반응온도, 농도)에 대하여 반응량을 최대화 할 수 있는 조건을 찾으려고 한다.

1.2 Response Optimizer의 실행
① Stat > DOE > Factorial > Response Optimizer... 를 선택한다.

② Selected란에 반응량을 을 이용하여 옮긴다.





③ Setup...을 클릭한다.




Target : 반응변수의 목표값을 입력한다. 여기서는 9로 입력한다
Lower : 반응변수의 규격 하한값을 입력한다. 여기서는 7로 입력한다.
Upper : 반응변수의 규격 상한값을 입력한다. 여기서는 11로 입력한다.
Goal   : 반응변수가 추구하는 특성에 따라 아래 그림과 같이 세가지 종류가 있다.
              "Minimize", "Target" 및 "Maximize"의 세가지 형태이다.
              "Minimize"은 반응변수의 값을 최소화 시키고자 하는 것이고,
              "Target"은 찾고자 하는 반응변수의 목표값이 특정한 값으로 주어 졌을 때 사용하며               "Maximize"은 반응변수의 값을 최대로 하고자 할 때 사용한다.





만족도(Desirability) : 반응변수가 Goal에 얼마나 잘 만족하는 지를 나타내는 측도로서, 완벽하게 만족하면 1이 되고 덜 만족할수록 0에 가까워 지는 값을 갖는다. 우리는 반응표면 분석을 통해서 여러 개의 반응 변수들의 종합적인 만족도(Overall Desirability)가 1에 가까워지도록 하는 인자들의 수준조합을 찾고자 하는 것이다.

가중치(Weight) : 각 Goal의 형태별로 만족도 함수(Desirability Function)의 모양을 정의 하며, 0.1에서 10 사이의 값을 가질 수 있다.
- 가중치가 1보다 클 경우 : 반응변수가 Target에서 멀어질수록 만족도(Desirability) 값이 급격히 작아지게 된다. 이러한 반응변수는 Target 값과 정확하게 일치하는 것이 최적의 상태라는 것을 의미한다.
- 가중치가 1인 경우 : 반응변수가 Target에서 멀어질수록 만족도 값은 선형적으로 감소하게 된다.
- 가중치가 1보다 작은 경우 : 반응변수가 Target에서 멀어질수록 만족도 값은 완만하게 감소한다. 이러한 반응변수는 Target 값과 정확히 일치하지 않아도 거의 최적의 상태를 유지할 수 있다는 것을 의미한다.
중요도(Importance) : 반응 변수들간의 상대적인 중요도를 나타낸다. 이 값은 0.1에서 10 사이의 값을 가질 수 있으며, 만약 모든 반응변수가 동등하게 중요하면 모두 1의 값을 지정한다. 그렇지 않고 변수들간의 중요도에 차이가 있다면 서로 다른 값을 지정한다. 이 값이 클수록 더 중요하며 작을수록 중요도가 낮다는 것을 의미한다.



④ Options...을 클릭한다.






이 옵션은 목표로 하는 반응변수의 값을 찾기 위한 각 인자들의 수준의 시작 값을 지정하는 기능이다. 모든 최적 해를 구하는 방법론은 처음에 각 인자들에 대한 초기값이 있어야 목표로 하는 반응변수의 목표 값을 찾아 갈 수 있다. 만일 우리가 어느 정도 최적 해에 대한 정보를 가지고 있다면 이들 정보를 이용하여 각 인자들의 초기값을 지정하면 된다. 그러나 대부분의 최적 해를 구하는 문제에서 이러한 정보를 얻기가 쉽지 않기 때문에 임의의 값으로 지정을 하면 된다. 본 예에서는 모든 인자의 수준의 초기 값을 각각 3.5, 55, 5.25로 주고 최적 해를 찾도록 하자.

Optimization plot을 선택하면 최적화 plot을 출력한다.
Store composite desirability values를 선택하면 반응 최적화 분석의 만족도 값을 워크시트에 저장한다.
Display local solutions를 선택하면 개별 반응값에 대한 최적조건을 출력한다.



Response Optimization 창에 나타난 결과




그래프에서 Targ는 목표값(Target)이고 d는 개별 반응변수의 만족도이며, D는 여러 반응변수를 함께 고려한 종합적인 만족도(Overall Desirability)를 나타낸다. 교반속도는 3.5, 반응온도 55.6302, 농도는 2.3922 일 때 반응량은 9로 목표값(Target)에 일치한다. 목표값에 대한 만족도도 1이다.


☞ 참 고

만약 새로운 목표값을 변경하고자 할 경우에는 그래프에서 직접 빨간선을 움직이면 개별 만족도가 변경되어 원하는 만족도를 찾을 수 있다. 또한, 각 인자에 최적수준에 클릭하면 새로운 창이 뜨는데 그 곳에 직접 입력해도 된다.

Session 창에 나타난 결과





그래프의 결과와 동일하며 쉽게 모든 사항을 확인할 수 있다.


Parameters에는 반응변수의 셋팅치 값을 나타낸다.
Starting Point는 각 인자의 최적알고리즘을 찾기 위한 초기 수준값이 나왔다.
Global solution는 목표값에 대한 인자의 최적조건을 나타낸다.
Predicted Responses는 개별 만족도이다.
Composite Desirability는 종합적인 만족도로 각 반응변수의 개별 만족도에 대해 가중 기하평균을 적용하여 합성된 만족도로 MINITAB에 의해 자동으로 계산된다. 본 예에서는 반응변수가 한 개이기 때문에 개별 만족도와 합성된 만족도가 동일하다


☞ 참고

MINITAB에서는 이 Session 창의 최적 해를 구하려면 반드시 인자의 변수 명을 영문으로 입력해야 된다. 만일 변수명을 한글로 입력하면 Response Optimization의 그래프 화면은 출력되지만 Session 창의 Response Optimization은 출력되지 않는다. 이것은 MINITAB이 영문환경에서 개발 되었기 때문에 한글을 인식하지 못하는 프로그램 상의 오류인 것으로 생각된다. 다시 한번 강조 하지만 인자들의 변수 명을 반드시 영문으로 지정할 것을 강조한다.
